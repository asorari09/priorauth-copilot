# PriorAuth Copilot

Prior-authorization decision-support demo built with Next.js 15, LangGraph, OpenAI extraction, Claude (citation synthesis + optional appeal drafts), Supabase policy RAG + inference cache, and Langfuse observability.

Outcomes are **code-determined** by deterministic rules and citation guardrails. Reasoning summaries are **template-generated by default** (`REASONING_MODE=template`); the LLM never chooses outcomes.

Production is deployed on Vercel with health check, optional `x-demo-key` for live runs, and SSE streaming.

## Public demo access

**No key required for the default demo.** Open the deployed app, keep **CASE-001** selected, leave **Run full agent pipeline** unchecked, and click **Run (instant cached demo)**. The UI replays a stored verified result from `data/presetDemoResults.json` (real CareSource citations and payer PDF URLs from a successful live run). The trace shows a single `stored_result (replay)` step — not a fake per-node animation.

| Mode | Demo key | LLM calls | Notes |
|------|----------|-----------|-------|
| CASE-001 cached (default) | Not required | None | Honest replay banner; zero cost |
| Live pipeline | `DEMO_KEY` required | Yes | Check **Run full agent pipeline** or pick a live-only scenario |
| Live-only scenarios | `DEMO_KEY` required | Yes | CASE-004, CASE-008, CASE-017, CASE-025 — no stored snapshot yet |

Set `DEMO_KEY` in Vercel env for live runs. Visitors trying live mode without the key receive HTTP 401.

Rebuild cached presets from Supabase (zero LLM):

```bash
npm run rebuild:presets   # writes data/presetDemoResults.json + data/liveOnlyPresets.json
```

Required env vars are listed in `.env.example`.

| Variable | Default | Used for |
|----------|---------|----------|
| `REASONING_MODE` | `template` | `template` = zero-cost deterministic reasoning; `llm` = Haiku prose via `ANTHROPIC_MODEL_REASONING` |
| `ANTHROPIC_MODEL_FAST` | `claude-haiku-4-5-20251001` | Citation synthesis + appeal drafts |
| `ANTHROPIC_MODEL_REASONING` | falls back to `ANTHROPIC_MODEL_FAST` | LLM reasoning only when `REASONING_MODE=llm` |

## Eval suite

### Architecture: who decides what

| Step | Engine | Role |
|------|--------|------|
| Extraction | OpenAI (`gpt-4o-mini`) | Structured clinical facts from free text |
| Rules check | TypeScript rules engine | Deterministic eligibility (zero LLM) |
| Policy RAG | pgvector + Haiku (cacheable) | Retrieve chunks; synthesize validated citations |
| **Outcome** | **TypeScript (`determineOutcomeConstraint`)** | **Always sets outcome** |
| **Reasoning prose** | **TypeScript template (default)** | Rules + citation summaries + unverified items |
| Appeal draft | Haiku (cacheable) | Deny-path letter only |
| Preset demo | `data/presetDemoResults.json` | CASE-001 instant replay from verified Supabase run; others live-only |

### Caching (deliberate production pattern)

| Layer | Key | Skips | Notes |
|-------|-----|-------|-------|
| Preset demo | `presetCaseId` | Entire graph | CASE-001 cached by default; `runLive: true` or live-only scenarios hit the graph |
| Citation synthesis | `sha256(CPT + sorted chunk_ids)` | Haiku citation call | Stored in Supabase `citation_cache` |
| Appeal draft | `sha256(case content hash)` | Haiku appeal call | Same table, `cache_kind=appeal_draft` |
| Eval freshness | `--no-cache` CLI flag | All inference cache reads/writes | Use when validating synthesis changes |

Caching is **explicit** — preset responses set `replay: true` and show a stored-result banner; live runs stream real graph node updates.

CI: eslint → tsc → vitest on every push/PR. **Eval job is `workflow_dispatch` only** and fails unless `decisionAccuracy=100`, `falseApproveRate=0`, `citationValidityRate=100` (full suite, `--no-cache`).

### Per-node cost model

Targets are **cost per case at equal eval accuracy**.

#### Baseline (pre-optimization): ~$0.045/case

| Node | Model | ~Cost/call |
|------|-------|------------|
| Citation synthesis | Sonnet 4.6 | $0.022 |
| Decision + reasoning | Sonnet 4.6 | $0.015 |
| Appeal draft | Sonnet 4.6 | $0.017 (deny only) |
| Extraction | gpt-4o-mini | $0.00009 |

#### Round 1 (Haiku + payload trim): ~$0.013/case projected

| Node | Model | ~Cost/call |
|------|-------|------------|
| Citation synthesis | Haiku 4.5 | ~$0.004 |
| Decision reasoning | Haiku 4.5 | ~$0.002 |
| Appeal draft | Sonnet 4.6 | ~$0.017 |
| Extraction | gpt-4o-mini | $0.00009 |

#### Round 2 (template reasoning + inference cache): target ~99% vs $0.045

| Scenario | ~Cost/case | What's billed |
|----------|------------|---------------|
| **Preset demo** | **$0** | Stored JSON snapshot; no API calls |
| **Warm cache hit** | **~$0.0001** | OpenAI extraction only |
| **First-run live (approve)** | **~$0.004** | Extraction + one Haiku citation synthesis |
| **First-run live (deny)** | **~$0.007** | Above + one Haiku appeal draft |
| **Repeat identical retrieval** | **~$0.0001** | Citations + appeal served from `citation_cache` |

**Projected reduction: ~$0.045 → ~$0.0001–0.007/case depending on cache warmth (~85–99%+ vs baseline).**

Round 2 levers:
1. **Template reasoning** — removes decision-node Claude call entirely (`REASONING_MODE=template`)
2. **Citation cache** — Supabase `citation_cache` keyed by CPT + sorted chunk IDs
3. **Appeal on Haiku + cache** — deny path uses `ANTHROPIC_MODEL_FAST` with content-hash cache
4. **Preset demo cache** — CASE-001 from verified Supabase `cases` row (`npm run rebuild:presets`)

Anthropic `cache_control` on system prompts remains wired but contributes ~$0 at current one-line prompt sizes.
